### Импорт OVA/OVF в консоли
Команда импорта ВМ из файлов OVF и VMDK:
```bash
# qm importovf <VMID> /path/to/vm.ovf <целевое хранилище>
```
Процедура импорта ВМ:
Скопировать файлы ovf и vmdk на хост PVE (в примере haos_ova-15.2.ovf и haos_ova-15.2.vmdk).
Создать новую ВМ, используя имя ВМ, информацию о ЦП и памяти из файла конфигурации OVF, и импортировать диски в хранилище local-lvm:
```bash
# qm importovf 999 haos_ova-15.2.ovf local-lvm
```
Примечание
Если вместо файлов ovf и vmdk есть файл ova, его необходимо предварительно распаковать, например:
```bash
$ tar -xvf haos_ova-15.2.ova
```
> Примечание
> - Сеть необходимо настроить вручную.

### Для возможности импорта ВМ из файлов OVA/OVF в веб-интерфейсе необходимо файловое хранилище с типом содержимого import. 
Для хранилища с таким типом содержимого будет создан каталог import, в который можно загрузить файлы OVA (архивный формат) или набор файлов OVF + VMDK/RAW (в плоской структуре, без подкаталогов).<br>
OVA-файлы можно также загрузить в хранилище напрямую через веб-интерфейс:<br>
В дереве ресурсов выбрать выбрать хранилище, с настроенным типом содержимого import.<br>
Перейти на вкладку Импорт.<br>
Нажать кнопку Отправить, если нужно загрузить файл с локального диска, или кнопку Скачать по URL-адресу, если нужно скачать файл с сервера.<br>
Для локальной загрузки: выбрать файл и нажать кнопку Отправить:<br>
Загрузка файла OVA в каталог import с локального диска<br>
Для загрузки с сервера: указать URL-адрес и нажать кнопку Скачивание:<br>
Загрузка файла OVA в каталог import с сервера<br>
Процедура импорта ВМ:<br>
В дереве ресурсов выбрать выбрать хранилище, и перейти на вкладку Импорт.<br>
Выбрать файл OVA и запустить мастер импорта, нажав кнопку Импорт:<br>
Запуск мастера импорта<br>
В открывшемся окне настроить параметры ВМ:<br>
на вкладке Общее указать базовые параметры ВМ: имя ВМ, ID, ресурсы;<br>
Импорт ВМ. Вкладка Общее<br>
на вкладке Дополнительно можно выбрать целевые хранилища для дисков ВМ и изменить сетевые настройки:<br>
Импорт ВМ. Вкладка Дополнительно<br>
на вкладке Итоговая конфигурация можно проверить сводку настроек перед началом импорта.<br>
Нажать кнопку Импорт.<br>
> Примечание
> - Для OVA-файлов требуется дополнительное место для временного извлечения образов. По умолчанию используется исходное хранилище, но можно указать отдельное рабочее хранилище для распаковки.<br>
> Примечание
> -Поскольку структура и содержимое файлов OVA/OVF не всегда хорошо определяются, может потребоваться вручную указать некоторые гостевые настройки.
